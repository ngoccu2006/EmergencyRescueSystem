# Rescue System - Domain Entities Documentation (Updated with Identity & Roles)

## 1. Overview

This system uses **ASP.NET Identity** with **Role-Based Authorization**.

Instead of separate entities like Citizen, Rescuer, Dispatcher, Commander:

* There is **only one User entity**
* Roles are used to determine behavior and permissions

---

## 2. Identity Model

## 2.1 User (ApplicationUser)

Represents all users in the system.

### Fields

* id: Guid
* email: string
* username: string
* fullname: string
* avatarUrl: string
* isActive: boolean
* createdAt: DateTime
* updatedAt: DateTime

### Notes

* Inherits from: `IdentityUser<Guid>`
* Used for authentication and authorization
* All roles are assigned to this entity

---

## 2.2 Roles

Roles define what a user can do in the system.

### Available Roles

* **Citizen**
* **Rescuer**
* **Dispatcher**
* **Commander**

---

## 3. Role Responsibilities

### 3.1 Citizen

* Create rescue requests
* View request status
* Manage personal profile

---

### 3.2 Rescuer

* Receive assigned missions
* Update mission progress
* Submit reports

---

### 3.3 Dispatcher

* Handle incoming requests
* Assign missions to rescue teams
* Manage request flow

---

### 3.4 Commander

* Monitor system activities
* Approve/reject requests
* Review reports and analytics

---

## 10. Relationships Summary

* A **User (Citizen)** creates multiple **Requests**
* A **User (Dispatcher)** handles and assigns requests
* A **User (Rescuer)** participates in **Rescue Teams**
* A **Mission** is created from a **Request**
* A **Mission** is handled by a **Rescue Team**
* A **Mission** generates **Reports**
* A **User (Commander)** oversees the system

---

## 11. Authorization Model

* Uses **ASP.NET Identity Roles**
* Access control via `[Authorize(Roles = "...")]`

### Example

```csharp
[Authorize(Roles = "Dispatcher")]
```

```csharp
[Authorize(Roles = "Commander")]
```

---

## 12. Notes for Implementation

* Use `UserManager<ApplicationUser>` for user operations
* Use `RoleManager<IdentityRole<Guid>>` for roles
* Do NOT create separate tables for Citizen, Rescuer, etc.
* Use role checks instead

---

## 13. Summary

* Single User entity
* Role-based behavior
* Clean and scalable design
* Fully compatible with ASP.NET Identity
